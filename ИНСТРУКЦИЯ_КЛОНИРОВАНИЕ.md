# Как клонировать репозиторий GitHub - Инструкция для новичков

## Что такое клонирование?
Клонирование - это создание локальной копии репозитория с GitHub на ваш компьютер.

## Предварительные требования

### 1. Установите Git
Если Git еще не установлен:
- Скачайте с официального сайта: https://git-scm.com/download/win
- Установите, используя настройки по умолчанию
- После установки откройте PowerShell или командную строку

### 2. Проверьте установку Git
Откройте PowerShell и выполните:
```powershell
git --version
```
Если видите версию (например, `git version 2.42.0`), значит Git установлен правильно.

## Клонирование репозитория

### Шаг 1: Выберите папку для проекта
Откройте PowerShell и перейдите в папку, где хотите сохранить проект:
```powershell
cd C:\Users\ВашеИмя\Documents
# или
cd C:\Users\ВашеИмя\source\repos
```

### Шаг 2: Клонируйте репозиторий
Выполните команду:
```powershell
git clone https://github.com/Tenzer1231/StringTop.git
```

### Шаг 3: Перейдите в папку проекта
```powershell
cd StringTop
```

### Шаг 4: Проверьте содержимое
```powershell
dir
# или
ls
```

## Что дальше?

### Открыть проект в Visual Studio:
1. Откройте Visual Studio
2. File → Open → Project/Solution
3. Найдите папку `StringTop`
4. Откройте файл `П412NEW.sln`

### Или через проводник:
1. Откройте папку `StringTop` в проводнике Windows
2. Дважды кликните на `П412NEW.sln`

## Полезные команды Git для новичков

### Проверить статус репозитория:
```powershell
git status
```

### Посмотреть историю коммитов:
```powershell
git log
```

### Обновить локальную копию (если в репозитории появились изменения):
```powershell
git pull
```

## Альтернативный способ: через GitHub Desktop

Если командная строка кажется сложной, можно использовать GitHub Desktop:

1. Скачайте GitHub Desktop: https://desktop.github.com/
2. Установите и войдите в свой аккаунт GitHub
3. Нажмите "Clone a repository from the Internet"
4. Выберите репозиторий `Tenzer1231/StringTop`
5. Выберите папку для сохранения
6. Нажмите "Clone"

## Частые проблемы

### Ошибка "git: command not found"
- Git не установлен или не добавлен в PATH
- Переустановите Git и убедитесь, что выбрали опцию "Add Git to PATH"

### Ошибка доступа
- Убедитесь, что репозиторий публичный
- Проверьте правильность URL

### Долгое клонирование
- Это нормально для больших репозиториев
- Проверьте интернет-соединение
